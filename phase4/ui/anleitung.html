Anleitung: LLM Control Deck

Diese Seite fasst den empfohlenen Ablauf zusammen und erklaert typische Statusmeldungen im UI.

Control Deck Workspaces Modelle HW Ueberwachung Restart Hilfe

1) Schnellstart

  1. Login mit User und Rolle ausfuehren (typisch: clemi / admin).
  2. Workspace auswaehlen oder ueber Dateisystem neu anlegen.
  3. Projekt und Top-K setzen (z. B. mim-llm und 5).
  4. Bei Modellwechsel auf Modelle pruefen klicken.
  5. Prompt senden und Quellen/Transparenz rechts kontrollieren.

2) Modellstatus richtig lesen

NUTZBAR Modell antwortet innerhalb der Live-Pruefung.

NICHT NUTZBAR Modell war nicht erreichbar oder hat keinen gueltigen Response geliefert.

  • Bei nicht nutzbaren Modellen zeigt das UI automatisch eine Alternative.
  • Wenn nur Browser-Fetch fehlschlaegt, wird der Grund als Netzwerkproblem ausgewiesen.
  • Router-Endpunkt fuer Modellpruefung: http://127.0.0.1:4000/v1/chat/completions

3) Smoke-Tests und Button-Farben

  • Mit Smoke-Tests ausfuehren werden Kernfunktionen geprueft.
  • Buttons werden nach dem Lauf markiert: gruen = Test ok, rot = Test fehlgeschlagen.
  • Die Detailergebnisse stehen im Bereich Smoke-Tests rechts.

4) Typische Fehlerbilder

  • NetworkError: Browser kann API/Router nicht erreichen. Pruefe Ports 4000/4100 und lokale Firewall.
  • Model not found: Modell nicht lokal vorhanden. Pruefe mit ollama list und ggf. ollama pull ....
  • All model attempts failed: Primaermodell und Fallbacks fehlgeschlagen. Router-Config und Service-Status pruefen.
