Beispielprompts (20, kompakt kategorisiert)

Praktische Prompt-Muster fuer unterschiedliche Aufgabenklassen, inspiriert von verbreiteten Prompting-Patterns aus Produktivumgebungen.

1) Planung / Projektsteuerung4 Prompts

Erstelle einen priorisierten 90-Tage-Plan fuer das Projekt mim-llm mit Meilensteinen, Risiken und klaren Exit-Kriterien je Phase.

Gib mir einen konkreten Umsetzungsplan in 5 Schritten fuer [Ziel], inklusive Aufwand (S/M/L), Abhaengigkeiten und erwarteten Nebenwirkungen.

Fasse den aktuellen Stand in maximal 10 Punkten zusammen und nenne danach die 3 wichtigsten naechsten Entscheidungen.

Erstelle ein Entscheidungs-Memo mit Optionen A/B/C, Trade-offs, Risiken und einer klaren Empfehlung mit Begruendung.

2) Analyse / Recherche / Reasoning4 Prompts

Analysiere dieses Problem systematisch: nenne Annahmen, Unsicherheiten, moegliche Fehlinterpretationen und die 3 robustesten Schlussfolgerungen.

Vergleiche [Option 1] vs [Option 2] fuer mein Setup in einer Tabelle: Performance, Betriebskosten, Risiken, Wartbarkeit, Komplexitaet.

Finde die wahrscheinlichsten Root Causes fuer [Symptom] und gib mir einen Diagnoseplan in sinnvoller Reihenfolge.

Was habe ich bei dieser Fragestellung wahrscheinlich uebersehen? Gib mir eine Red-Team-Liste mit 8 kritischen Punkten.

3) Coding / Architektur4 Prompts

Refaktoriere den folgenden Code auf Lesbarkeit und Testbarkeit, ohne das Verhalten zu aendern. Erklaere die Aenderungen kurz.

Entwirf eine robuste API fuer [Use Case] inkl. Endpunkten, Request-/Response-Schema, Fehlercodes und Security-Hinweisen.

Schreibe einen minimalen, produktionsnahen Testplan fuer diese Aenderung: Unit, Integration, Failure-Path, Performance-Sanity.

Pruefe diesen Code wie in einer Senior-Code-Review: priorisiere Bugs, Sicherheitsrisiken und moegliche Regressionen.

4) Betrieb / SRE / Incident4 Prompts

Erstelle ein Incident-Runbook fuer [Fehlerbild] mit Sofortmassnahmen, Diagnosebaum und Rollback-Kriterien.

Definiere sinnvolle SLO/SLI fuer diesen Service und schlage dazu passende Alert-Thresholds inklusive Rauschen-Vermeidung vor.

Gib mir einen taeglichen Betriebscheck (Health, Logs, Ressourcen, Backups, Security) als kurze Checkliste mit Sollwerten.

Welche 5 Ausfallmodi sind in diesem Setup am wahrscheinlichsten, wie erkenne ich sie fruehzeitig und wie minimiere ich den Impact?

5) Kommunikation / Doku / Governance4 Prompts

Schreibe eine technische Aenderungsdoku im Stil: Ziel, Ausgangslage, Umsetzung, Risiken, Rollback, Validierung.

Formuliere ein Stakeholder-Update in 6 Bulletpoints: was erledigt, was offen, Risiken, naechste Schritte, ETA.

Erzeuge ein kurzes Governance-Review: Rollen, Berechtigungen, Audit-Trails, Datenhaltung, Backup-Policy, DR-Readiness.

Uebersetze diese technische Loesung in eine nicht-technische Version fuer Management mit Fokus auf Nutzen, Kosten und Risiken.
