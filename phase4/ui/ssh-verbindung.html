Wie funktioniert eine SSH-Verbindung?

SSH verbindet dich sicher mit einem Linux-Host ueber die Shell. Typisch ist Login per SSH-Key statt Passwort.

ssh user@server-ip

1) SSH-Key erzeugen (Client)

ssh-keygen -t ed25519 -C "dein-name@llm"
# Default-Pfad bestaetigen: ~/.ssh/id_ed25519

2) Public Key auf Server kopieren

ssh-copy-id user@server-ip
# alternativ: Inhalt von ~/.ssh/id_ed25519.pub in ~/.ssh/authorized_keys eintragen

3) Testen

ssh user@server-ip

Harter Basisschutz (Server)

  • In /etc/ssh/sshd_config Key-Login aktiv lassen.
  • PasswordAuthentication no setzen (wenn Key funktioniert).
  • PermitRootLogin no setzen.
  • SSH nur ueber VPN/Tailscale erlauben, wenn moeglich.
sudo systemctl restart ssh
Hinweis: Nach Aenderungen immer eine zweite SSH-Session offen lassen, damit du dich nicht aussperrst.

Praktische Befehle

# Port und Auth-Details anzeigen
ssh -v user@server-ip

# bestimmten Key nutzen
ssh -i ~/.ssh/id_ed25519 user@server-ip

# custom Port
ssh -p 2222 user@server-ip
