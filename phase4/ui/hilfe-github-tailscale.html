Wie fuege ich User aus GitHub hinzu?

In diesem Stack werden Berechtigungen lokal in der User-Verwaltung gepflegt. Ein GitHub-Username kann direkt als User-ID uebernommen werden, die Authentisierung bleibt aber weiterhin lokal/VPN-basiert.

Schritte

  1. GitHub-Handle ermitteln, z. B. octocat.
  2. In der Seite Rechte / User neuen User anlegen.
  3. Als Username den GitHub-Handle eintragen, Rolle waehlen, erlaubte Projekte setzen.
  4. Speichern. Der User kann sich danach mit genau diesem Namen einloggen.
Wichtig: Das ist eine Namenszuordnung (Identity Mapping), kein OAuth-Login gegen GitHub.

Warum Tailscale?

  • Tailscale baut ein privates, verschluesseltes Overlay-Netz fuer Admin/API/UI-Zugriffe.
  • Nur Geraete im Tailnet koennen interne Dienste (z. B. Port 4100/4173) erreichen.
  • ACLs in Tailscale steuern, welche Nutzergruppen welche Serverports erreichen duerfen.

Typischer Ablauf mit Tailscale

  1. User in Tailscale-Admin aufnehmen oder per Invite einladen.
  2. Client anmelden, Erreichbarkeit pruefen (Tailnet-IP / MagicDNS).
  3. Optional ACL-Regeln fuer API/UI auf Projektservern setzen.
  4. Danach User im LLM-Deck mit Rolle und Projekten anlegen.
Best Practice: Zugang auf zwei Ebenen steuern. 1) Netzwerkzugang via Tailscale. 2) Anwendungsrechte via Rollen in der Rechteverwaltung.
